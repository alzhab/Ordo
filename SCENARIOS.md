# TaskBot — Сценарии использования (BDD)

Документ описывает все пользовательские сценарии бота в формате Given/When/Then.
Предназначен для ревью (PM/BA) и как основа для написания интеграционных тестов.

## 1. Создание задачи

### SC-01 — Простая задача текстом
```
Given: пользователь отправляет текст "Купить молоко"
When:  Claude парсит намерение как create_task
Then:  бот показывает превью:
         Название: Купить молоко
         Категория: Общее
       с кнопками [✅ Создать] [✏️ Изменить] [❌ Отмена]
When:  пользователь нажимает ✅ Создать
Then:  задача сохраняется в SQLite (status=not_started)
  And: бот показывает "✅ Задача создана!" с кнопками [🤖 Предложить шаги] [📋 Открыть задачу]
```

### SC-02 — Задача с датой, категорией, приоритетом
```
Given: пользователь отправляет "Сдать отчёт по работе в пятницу, высокий приоритет"
When:  Claude парсит намерение как create_task
Then:  превью содержит:
         Название: Сдать отчёт
         Категория: Работа
         Срок: <ближайшая пятница>
         Приоритет: 🔴 Высокий
When:  пользователь нажимает ✅ Создать
Then:  задача сохраняется с due_date, category, priority
```

### SC-03 — Задача с подзадачами
```
Given: пользователь отправляет "Подготовить презентацию: сделать слайды, собрать данные, согласовать с командой"
When:  Claude парсит намерение как create_task с subtasks
Then:  превью показывает секцию "📋 Шаги (3):" со списком подзадач
When:  пользователь нажимает ✅ Создать
Then:  задача и 3 подзадачи сохраняются в SQLite
  And: кнопка после создания — только [📋 Открыть задачу] (без AI шагов)
  And: если Notion настроен — задача синхронизируется, затем подзадачи
       добавляются как Notion-блоки; каждой подзадаче сохраняется notion_block_id
```

### SC-04 — Отмена создания
```
Given: бот показывает превью задачи
When:  пользователь нажимает ❌ Отмена
Then:  сессия очищается
  And: бот показывает "❌ Отменено."
```

### SC-05 — Автоопределение статуса waiting при создании
```
Given: пользователь отправляет "Записался на приём к врачу на пятницу"
When:  Claude парсит намерение как create_task
Then:  превью содержит:
         Статус: ⏸ В ожидании
         Причина: приём у врача
         До: <ближайшая пятница>
When:  пользователь нажимает ✅ Создать
Then:  задача сохраняется с status=waiting, waiting_reason, waiting_until
```

```
Признаки автоопределения waiting:
  "я записался", "я заказал", "я отправил", "я договорился",
  "жду", "должно прийти", "должны позвонить"
```

## 2. Пакетное создание задач

### SC-06 — Два и более независимых задач в одном сообщении
```
Given: пользователь отправляет "Купить молоко, позвонить врачу и записаться на стрижку"
When:  Claude парсит намерение как create_tasks_batch (3 задачи)
Then:  бот показывает слайдер — первая задача:
         📝 Задача 1 из 3
         Название: Купить молоко
       с кнопками [✅ Создать] [⏭ Пропустить] [❌ Отмена]
When:  пользователь нажимает ✅ Создать
Then:  первая задача сохраняется
  And: бот показывает вторую задачу (Задача 2 из 3)
When:  пользователь нажимает ⏭ Пропустить
Then:  вторая задача не создаётся
  And: бот показывает третью задачу (Задача 3 из 3)
When:  пользователь нажимает ✅ Создать
Then:  третья задача сохраняется
  And: бот показывает итог: "✅ Создано 2 задачи (пропущено: 1)"
       с кнопками-ссылками на каждую созданную задачу
```

### SC-07 — Отмена в середине слайдера
```
Given: слайдер показывает задачу N из M
When:  пользователь нажимает ❌ Отмена
Then:  сессия очищается
  And: бот показывает "❌ Отменено."
  And: уже созданные задачи (до нажатия Отмена) остаются в БД
```

### SC-08 — Одна задача в batch (деградация)
```
Given: пользователь отправляет текст, Claude возвращает create_tasks_batch с 1 задачей
Then:  бот показывает обычное превью (SC-01) вместо слайдера
```

## 3. Просмотр списка задач

### SC-09 — Команда /tasks
```
Given: пользователь отправляет /tasks
Then:  бот показывает список задач со статусом in_progress
  And: заголовок "🔄 В работе (N):"
  And: строки фильтров:
         Строка 1: [📁 Категория] [📊 Статус: 🔄 В работе ×] [📋 План]
         Строка 2: [📦 Архив] [🔍 Поиск]
  And: до 15 задач, каждая — inline-кнопка (жёсткое ограничение, кнопки "ещё" нет)
```

### SC-10 — Фильтр по статусу
```
Given: бот показывает список задач
When:  пользователь нажимает [📊 Статус] или [🔄 В работе ×]
Then:  бот показывает подменю статусов:
         [🔄 В работе] [⬜ Не начато]
         [⏸ В ожидании] [✅ Выполненные]
         [🔁 Все активные]
         [◀️ Назад]
When:  пользователь выбирает "⏸ В ожидании"
Then:  список обновляется — только waiting задачи
  And: сортировка: задачи с waiting_until (включая просроченные) — по дате ASC,
       задачи без waiting_until — в конце
       (просроченные визуально отмечены ⚠️, но не выносятся отдельной группой)
  And: кнопка статуса показывает "⏸ В ожидании ×"
```

### SC-11 — Фильтр «Все активные»
```
Given: пользователь выбирает "🔁 Все активные" в подменю статусов
Then:  список показывает все задачи кроме done и deleted
  And: включает задачи со статусами not_started, in_progress, waiting
  And: кнопка статуса показывает "📊 Статус" (без активного фильтра)
```

### SC-12 — Фильтр по категории
```
Given: бот показывает список задач
When:  пользователь нажимает [📁 Категория]
Then:  бот показывает список доступных категорий кнопками
When:  пользователь выбирает категорию "Работа"
Then:  список обновляется — только задачи категории "Работа"
  And: кнопка категории показывает "📁 Работа ×"
When:  пользователь нажимает "📁 Работа ×"
Then:  фильтр категории сбрасывается
```

### SC-13 — Поиск по задачам
```
Given: бот показывает список задач
When:  пользователь нажимает [🔍 Поиск]
Then:  бот просит ввести поисковый запрос текстом
When:  пользователь отправляет "отчёт"
Then:  список обновляется — задачи с "отчёт" в названии или описании
  And: кнопка поиска показывает "🔍 "отчёт" ×"
```

### SC-14 — Фильтр архивных задач
```
Given: бот показывает список задач
When:  пользователь нажимает [📦 Архив]
Then:  в список включаются задачи из архивных планов
  And: кнопка архива показывает "📦 Архив ×"
```

### SC-15 — Пустой список
```
Given: у пользователя нет задач с активным фильтром
Then:  бот показывает "🔄 В работе (0)\n\nНичего не найдено."
  And: кнопки фильтров остаются доступны
```

## 4. Просмотр карточки задачи

### SC-16 — Открытие карточки из списка
```
Given: бот показывает список задач
When:  пользователь нажимает на задачу в списке
Then:  сообщение со списком удаляется
  And: бот показывает карточку задачи:
         📌 Название
         Статус, Категория, Приоритет, Срок, Описание, План
         Шаги: N/M (если есть)
         Ссылка на Notion (если привязана)
  And: кнопки: [⬜ Вернуть] [🔄 В работу] [✅ Готово] [⏸ В ожидании]
               [📋 Шаги] [✏️ Изменить]
               [🗑 Удалить] [◀️ К списку]
```

### SC-17 — Карточка waiting задачи
```
Given: задача имеет status=waiting
Then:  карточка дополнительно показывает:
         Причина: <waiting_reason>
         До: <waiting_until> (с ⚠️ если дата просрочена)
  And: кнопки статуса не содержат [⏸ В ожидании] (уже активен)
```

### SC-18 — Возврат к списку из карточки
```
Given: пользователь просматривает карточку задачи (открытой из /tasks)
When:  нажимает [◀️ К списку]
Then:  бот показывает список задач с последним активным фильтром
```

### SC-19 — Возврат к плану из карточки
```
Given: пользователь открыл задачу из списка задач плана
When:  нажимает [◀️ К плану]
Then:  бот показывает список задач этого плана
```

## 5. Редактирование задачи до создания

### SC-20 — Изменение поля через меню
```
Given: бот показывает превью задачи
When:  пользователь нажимает ✏️ Изменить
Then:  бот показывает меню полей:
         [Название] [Описание] [Категория] [Приоритет] [Дата] [План]
When:  пользователь выбирает "Название"
Then:  бот просит ввести новое название
When:  пользователь вводит новое название
Then:  превью обновляется с новым названием
  And: кнопки [✅ Создать] [✏️ Изменить] [❌ Отмена] снова активны
```

### SC-21 — Изменение категории через кнопки
```
Given: пользователь в меню редактирования нажимает "Категория"
Then:  бот показывает список категорий кнопками (с кнопкой ◀️ Назад)
When:  пользователь выбирает категорию
Then:  превью обновляется с новой категорией
```

### SC-22 — Изменение плана через список
```
Given: пользователь в меню редактирования нажимает "План"
Then:  бот показывает список активных планов + [❌ Без плана]
When:  пользователь выбирает план
Then:  превью обновляется с выбранным планом
```

## 6. Редактирование задачи после создания

### SC-23 — Открытие меню редактирования сохранённой задачи
```
Given: пользователь просматривает карточку задачи
When:  нажимает ✏️ Изменить
Then:  бот показывает меню полей:
         [Название] [Описание]
         [Категория] [Приоритет]
         [Дата срока] [План]
       (если статус waiting — дополнительно:)
         [⏸ Причина] [⏸ Дата ожидания]
         [◀️ Назад]
```

### SC-24 — Редактирование поля текстом
```
Given: пользователь выбрал поле для редактирования (например, Название)
Then:  бот показывает текущее значение и просит ввести новое
When:  пользователь вводит новый текст
Then:  задача обновляется в SQLite
  And: Notion sync обновляет поле (если задача привязана к Notion)
  And: бот показывает обновлённую карточку задачи
```

### SC-25 — Редактирование даты
```
Given: пользователь выбрал поле "Дата срока"
When:  пользователь вводит "25 марта" / "через неделю" / "2026-04-01"
Then:  дата парсится через parseFlexibleDate
  And: задача обновляется с due_date в формате ISO
```

## 7. Смена статуса задачи

### SC-26 — Перевод в работу
```
Given: задача имеет status=not_started
When:  пользователь нажимает [🔄 В работу] в карточке
Then:  задача обновляется до status=in_progress
  And: карточка обновляется inline (без перезагрузки)
  And: Notion sync обновляет Status
```

### SC-27 — Отметить выполненной
```
Given: задача имеет любой статус кроме done
When:  пользователь нажимает [✅ Готово]
Then:  задача обновляется до status=done
  And: задача исчезает из списка с фильтром in_progress/not_started/waiting
  And: Notion sync обновляет Status
```

### SC-28 — Выход из статуса waiting
```
Given: задача имеет status=waiting
When:  пользователь нажимает [🔄 В работу]
Then:  задача обновляется: status=in_progress, waiting_reason=null, waiting_until=null
  And: Notion sync обновляет Status=In Progress, Waiting Reason="", Waiting Until=null
When:  пользователь нажимает [⬜ Вернуть]
Then:  задача обновляется: status=not_started, waiting_reason=null, waiting_until=null
  And: Notion sync обновляет Status=Not Started, Waiting Reason="", Waiting Until=null
```

### SC-29 — Смена статуса голосом
```
Given: существует задача "Купить молоко"
When:  пользователь отправляет "задача купить молоко готова"
Then:  Claude парсит как manage_task, action=update_status, status=done
  And: fuzzy-поиск находит задачу
  And: задача обновляется до done
  And: бот подтверждает с кнопкой [Открыть]
```

## 8. Статус «В ожидании»

### SC-30 — Перевод задачи в waiting через кнопку (двухшаговый диалог)
```
Given: пользователь в карточке задачи нажимает [⏸ В ожидании]
Then:  бот спрашивает "Чего ждёшь? Опиши причину:" + [Пропустить]
When:  пользователь вводит "жду звонка от врача"
Then:  бот проверяет: есть ли дата в тексте причины
  If:  дата найдена → финализирует waiting без шага "до какой даты"
  If:  дата не найдена → спрашивает "До какой даты ждёшь?" + [Пропустить]
When:  пользователь вводит "в пятницу"
Then:  задача обновляется: status=waiting, waiting_reason, waiting_until
  And: карточка обновляется с полями ожидания
  And: Notion sync обновляет Status=Waiting, Waiting Reason, Waiting Until
```

### SC-31 — Автоизвлечение даты из причины
```
Given: бот спрашивает причину ожидания
When:  пользователь вводит "заказ на WB придёт 25 марта"
Then:  extractDateFromText находит "25 марта" → waiting_until = "2026-03-25"
  And: бот пропускает шаг "до какой даты" (дата уже есть)
  And: задача финализируется сразу
```

### SC-32 — Пропуск причины и даты
```
Given: бот спрашивает причину ожидания
When:  пользователь нажимает [Пропустить]
Then:  бот спрашивает "До какой даты?"
When:  пользователь снова нажимает [Пропустить]
Then:  задача обновляется: status=waiting, waiting_reason=null, waiting_until=null
```

### SC-33 — Просроченная дата ожидания
```
Given: задача имеет waiting_until = дата в прошлом
Then:  в карточке задачи поле "До:" отображается как "⚠️ 15 мар 2026"
  And: в списке задач с фильтром "В ожидании" задача сортируется по дате ASC
       (просроченные окажутся первыми автоматически, т.к. их дата < остальных)
  And: визуальный маркер ⚠️ — единственный способ отличить просроченные
       (отдельная сортировочная группа не создаётся)
```

## 9. Удаление задачи

### SC-34 — Удаление через кнопку в карточке
```
Given: пользователь в карточке задачи нажимает [🗑 Удалить]
Then:  бот показывает "Удалить задачу?" + [Да, удалить] [Отмена]
When:  пользователь подтверждает
Then:  задача обновляется до status=deleted (soft delete — физически не удаляется)
  And: Notion sync вызывает updateTaskStatus(pageId, 'deleted') →
       обновляет поле Status на "Deleted" в базе задач Notion
       (NB: для планов используется archiveNotionPage — страница архивируется;
        для задач — только меняется поле Status)
  And: задача исчезает из всех списков (кроме архивных запросов)
```

### SC-35 — Удаление голосом
```
Given: существует задача "Купить молоко"
When:  пользователь отправляет "удали задачу купить молоко"
Then:  Claude парсит как manage_task, action=delete
  And: бот запрашивает подтверждение удаления
```

## 10. Подзадачи (шаги)

### SC-36 — Просмотр шагов
```
Given: задача имеет подзадачи
When:  пользователь нажимает [📋 Шаги] в карточке
Then:  бот показывает список шагов:
         ☐ Шаг 1  ✏️  🗑
         ☑ Шаг 2  ✏️  🗑
       с кнопками [➕ Добавить шаг] [🤖 AI шаги] [◀️ К задаче]
```

### SC-37 — Тогл шага
```
Given: бот показывает список шагов
When:  пользователь нажимает на шаг "☐ Шаг 1"
Then:  шаг отмечается как выполненный (is_done=1)
  And: список обновляется: "☑ Шаг 1"
  And: Notion sync обновляет чекбокс по notion_block_id
```

### SC-38 — Добавление шага вручную
```

### SC-68 — Голосовой перевод задачи в waiting (set_waiting)
```
Given: существует задача "Купить запчасти"
When:  пользователь отправляет "задача купить запчасти в ожидании, жду доставку до 25 марта"

         waiting_reason="жду доставку", waiting_until="2026-03-25"
  And: fuzzy-поиск находит задачу
  And: normalizeWaiting применяется (извлекает дату из причины если waiting_until=null)
  And: задача обновляется: status=waiting, waiting_reason, waiting_until
  And: бот подтверждает "⏸ Купить запчасти — в ожидании" с кнопкой [Открыть]
  And: Notion sync обновляет Status=Waiting, Waiting Reason, Waiting Until
```

```
Вариант без даты:
Given: пользователь отправляет "задача X в ожидании, жду ответа от клиента"

       waiting_until=null
```

### SC-69 — Сохранение фильтра при переходе в карточку и обратно
```
Given: пользователь установил фильтр категория="Работа" в списке задач

  And: фильтры хранятся в памяти (taskFilters map) и сбрасываются только при
       явном нажатии × или перезапуске бота
```

### SC-70 — /today и задачи с waiting_until = сегодня
```
Given: есть задача A с due_date = сегодня и статусом in_progress
  And: есть задача B с waiting_until = сегодня и статусом waiting

  And: задача B НЕ отображается (/today фильтрует только по due_date)

  /tasks → фильтр "В ожидании" → задача B будет отмечена ⚠️ (дата истекла)
