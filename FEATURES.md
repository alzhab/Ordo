# Ordo — Карта функционала

> Справочник для разработчика: что есть в боте, как вызвать, пример.

---

## 1. Создание задач

### Одиночная задача

Голос или текст в любом свободном формате. Claude парсит намерение и заполняет поля.

**Как вызвать:** отправить текст / голос / `/add <текст>`

**Что определяется автоматически:**
- Название задачи
- Категория (из существующих или новая)
- Дедлайн (абсолютный или относительный — "в пятницу", "через неделю")
- Статус `waiting` (если фраза содержит "записался", "заказал", "жду", "договорился")
- `waiting_reason` и `waiting_until`
- Подзадачи (если перечислены шаги)

**Пример:**
```
Ты:   Купить молоко сегодня вечером, категория дом
Бот:  [превью задачи]
      [✅ Создать] [✏️ Изменить] [❌ Отмена]
```

```
Ты:   Записался к стоматологу на пятницу
Бот:  Создаю задачу "Визит к стоматологу"
      Статус: ⏸ Ожидание
      Жду: приём у стоматолога
      До: пятница
```

---

### Несколько задач сразу (batch)

Если в сообщении перечислено 2+ независимых задач — бот создаёт их по одной через слайдер.

**Как вызвать:** перечислить задачи через запятую / союзы "и", "плюс", "также"

**Пример:**
```
Ты:   Купить молоко, позвонить врачу и записаться на стрижку
Бот:  Задача 1 из 3: "Купить молоко"
      [✅ Создать] [⏭ Пропустить]
```

---

### Задача-ожидание (waiting)

Задачи где пользователь уже что-то сделал и ждёт внешнего события.

**Два типа waiting:**
- **Тип А** — `waiting_until` задана → бот не трогает пока дата не пришла
- **Тип Б** — `waiting_until` не задана → бот поднимает в вечернем разборе через N дней

**Пример:**
```
Ты:   Заказал вешалки на WB, придут 22 марта
Бот:  ✅ Создано: "Купить вешалки"
      Статус: ⏸ Ожидание до 22 марта

Ты:   Отправил резюме в компанию X
Бот:  ✅ Создано: "Резюме в компанию X"
      Статус: ⏸ Ожидание ответа
```

---

### Большая цель → план с задачами (suggest_plan)

Claude разбивает цель на 3–5 задач с подзадачами и предлагает создать план.

**Как вызвать:** фразы "хочу", "помоги спланировать", "подготовиться к", "организовать"

**Пример:**
```
Ты:   Хочу подготовиться к свадьбе через 4 месяца
Бот:  📋 План: Свадьба

      1. Выбрать дату и площадку
         → Составить список площадок
         → Согласовать дату с семьёй
         → Забронировать

      2. Гости и приглашения
         ...

      [✅ Создать план] [❌ Отмена]
```

## 2. Просмотр задач

### Список задач

**Команда:** `/tasks`

**Фильтры (через кнопки):**
- По статусу: Все / В работе / Не начаты / Ожидание / Выполнено
- По категории
- По плану
- Поиск по тексту
- Архивные задачи (done / deleted)

**Команда:** `/today` — задачи с дедлайном на сегодня

---

### Детальный просмотр задачи

Нажать на задачу в списке → открывается карточка с:
- Статус, категория, план, дедлайн
- Описание
- Подзадачи (шаги)
- Кнопки редактирования

---

## 3. Управление задачей

### Изменение через голос/текст

**Как вызвать:** описать действие свободным текстом

| Фраза | Действие |
|---|---|
| "задача X в работу" | статус → todo |
| "отметь X выполненной" | статус → done |
| "удали задачу X" | статус → deleted (soft delete) |
| "перенеси X в план Y" | assign_plan |
| "поставь X на пятницу" | set_date |
| "задача X в ожидании, жду доставку" | set_waiting |
| "верни X в очередь" | статус → todo |

**Поиск задачи:** fuzzy-матч по названию. Если нашлось несколько — бот показывает список на выбор.

---

### Редактирование через кнопки

Из карточки задачи → `[✏️ Редактировать]`:
- Название
- Описание
- Дата дедлайна
- Категория
- Приоритет
- План
- Причина и срок ожидания (для waiting)

---

### Групповые операции (bulk)

**Как вызвать:** фразы с "все", "первые N", "последние N", "половина"

| Фраза | Действие |
|---|---|
| "все задачи готовы" | статус → done для всех |
| "первые 3 задачи в работу" | статус → todo для первых 3 |
| "все задачи по дому удали" | delete для категории Дом |
| "последние 2 задачи удали" | delete для последних 2 |

---

## 4. Подзадачи (шаги)

Один уровень подзадач — без рекурсии.

**Как открыть:** из карточки задачи → `[📋 Шаги]`

**Действия:**
- Отметить шаг выполненным / невыполненным
- Добавить шаг вручную
- Редактировать шаг
- Удалить шаг
- `[✨ Предложить шаги]` — AI генерирует 3–5 шагов
  - `[Заменить]` — заменить существующие
  - `[Объединить]` — добавить недостающие (без дублей)

**Создание через голос:** перечислить шаги при создании задачи — попадут в subtasks автоматически.

---

## 5. Планы

### Список планов

**Команда:** `/plans`

Каждый план показывает прогресс: количество задач выполнено / всего.

---

### Создание плана

**Вариант 1 — пустой план:**
```
Ты:   Создай план Ремонт
Бот:  ✅ План "Ремонт" создан
```

**Вариант 2 — через кнопку:** `/plans` → `[+ Новый план]`

**Вариант 3 — AI разбивка цели:** (см. suggest_plan выше)

---

### Управление планом

| Действие | Как |
|---|---|
| Просмотр задач плана | `/plans` → выбрать план |
| Архивировать | из карточки плана → `[Архивировать]` |
| Редактировать название/описание | из карточки → `[✏️]` |
| Удалить (только план) | архив → удалить → оставить задачи |
| Удалить (с задачами) | архив → удалить → удалить задачи |
| Восстановить из архива | архив → `[Восстановить]` |

## 6. Проактивный ассистент

### /morning — Утренний план

Claude анализирует все задачи и предлагает 3–5 приоритетных с объяснением.

**Команда:** `/morning`
**Автоотправка:** каждый день в `morning_time` (по умолчанию 09:00)

**Алгоритм:** дедлайны → истёкшие waiting → давно не трогавшиеся → контекст планов

**Пример:**
```
🌅 Доброе утро! На сегодня:

1. Позвонить маме насчёт даты свадьбы
   → блокирует выбор площадки

2. Проверить доставку вешалок
   → срок ожидания вышел вчера

[✅ Берусь] [✏️ Изменить список]
```

---

### /review — Вечерний разбор

Разбор зависших задач: истёкшие waiting, стоячие todo, задачи maybe.

**Команда:** `/review`
**Автоотправка:** каждый день в `evening_time` (по умолчанию 21:00)
**Максимум:** 5 задач за один разбор

**Типы задач в разборе:**
- `waiting` с истёкшей датой → "срок ожидания вышел, что делаем?"
- `waiting` без даты (3+ дней) → "пора напомнить?"
- `todo` без движения 7+ дней → "давно не двигается, что с этим?"
- `maybe` 7+ дней → "всё ещё думаешь об этом?"

**Кнопки по типу задачи:**
```
⏸ Задача ожидания (срок вышел):
   [▶️ Взять в работу] [⏸ Ещё жду] [❌ Закрыть]

📋 Стоячая задача:
   [▶️ Взять в работу] [💭 В maybe] [🗑 Удалить]

💭 Maybe задача:
   [✅ Беру в работу] [🗑 Удалить]

[⏭ Пропустить всё]
```

---

### /focus — Что делать прямо сейчас

Одна задача с объяснением почему именно она.

**Команда:** `/focus` (в любой момент)

**Пример:**
```
🎯 Сейчас делай:
Написать подрядчику по ремонту
→ 5-10 минут, ждёт ответа 3 дня

[✅ Сделал] [⏭ Другую задачу]
```

---

### /progress — Прогресс

**Команда:** `/progress`

**Показывает:**
- Выполнено сегодня / за неделю
- Прогресс по каждому плану (прогресс-бар █░)
- Задачи без движения 14+ дней

**Пример:**
```
📊 Прогресс

✅ Сегодня: 3 задачи
✅ За неделю: 11 задач

💍 Свадьба    ████░░░░  5/14
🏠 Ремонт     ██░░░░░░  3/10
```

## 7. Повторяющиеся напоминания

Не задачи — только напоминания по расписанию. После срабатывания сбрасываются на следующий цикл.

**Как создать:** голосом

**Типы расписания:**
- Конкретные дни недели: "каждый понедельник"
- Несколько дней: "вторник и четверг"
- Ежедневно: "каждый день в 8 утра"
- Рабочие дни: "рабочие дни в 9:30"
- День месяца: "1-го числа каждого месяца"

**Напоминание заранее:** "напомни за 30 минут" → уведомление приходит за 30 мин до события

**Примеры:**
```
Ты:   Каждый понедельник в 23:00 созвон, напомни за 30 минут
Бот:  ✅ Создал напоминание
      🔄 Созвон
      Каждый понедельник в 22:30

Ты:   Спортзал вторник и четверг в 18:00, напомни за час
Бот:  🔄 Спортзал
      вт, чт в 17:00

Ты:   Оплата аренды 1-го числа каждого месяца
Бот:  🔄 Оплата аренды
      1-го числа в 09:00
```

**Список и удаление:** `/reminders`

## 8. Настройки

### Через команду

**Команда:** `/settings`

**Разделы:**
- Notion интеграция (токен, database ID)
- Категории (просмотр, создание, удаление)
- Ошибки синхронизации Notion

---

### Через голос/текст (manage_settings)

| Фраза | Действие |
|---|---|
| "поставь план на 8 утра" | время утреннего плана → 08:00 |
| "вечерний разбор в 22:00" | время вечернего разбора → 22:00 |
| "не беспокой до завтра" | тихий режим до завтра 09:00 |
| "не беспокой до воскресенья" | тихий режим до воскресенья 09:00 |
| "выключи утренний план" | morning_enabled → false |
| "включи утренний план" | morning_enabled → true |
| "выключи вечерний разбор" | review_enabled → false |
| "выключи все напоминания" | всё отключить |
| "включи напоминания" | всё включить |

## 9. Категории

**Создание:**
```
Ты:   Создай категорию Спорт
Бот:  ✅ Категория "Спорт" создана
```

Или через `/settings` → Категории → `[+ Добавить]`

**Просмотр:**
```
Ты:   Какие у меня категории?
Бот:  [список категорий с количеством задач]
```

**Удаление:** только если нет привязанных задач — через `/settings` → Категории

---

## 10. Notion интеграция (опционально)

Если настроена — задачи автоматически синхронизируются с Notion.

**Настройка:** `/settings` → Notion → ввести токен и database ID

**Что синхронизируется:**
- Создание задачи → создаётся страница в Notion
- Изменение статуса, дедлайна, описания → обновляется в Notion
- Создание/изменение подзадач → обновляются блоки на странице
- Создание плана → создаётся страница в Notion Plans

**Ошибки синхронизации:** логируются, доступны через `/settings` → Ошибки синхронизации. Ошибки не блокируют основной флоу.

## Антиспам правила планировщика

- Максимум 2 автосообщения в день (утро + вечер)
- Тихий режим (`quiet_until`) — ничего не отправляется
- Пользователь неактивен 7+ дней — автосообщения не отправляются
- Дубли исключены через `notification_log` (персистентно, работает после перезапуска)

---

## Статусы задач

| Статус | Смысл |
|---|---|
| `todo` | Надо сделать (по умолчанию) |
| `waiting` | Жду внешнего события |
| `maybe` | Возможно когда-нибудь |
| `done` | Выполнено |
| `deleted` | Soft delete |

---

## Поддерживаемые форматы ввода

Каждая функция доступна тремя способами:
1. **Команда** — `/morning`, `/tasks`, `/focus`
2. **Текст** — свободная фраза, Claude парсит намерение
3. **Голос** — Groq Whisper транскрибирует, затем тот же флоу что и текст
