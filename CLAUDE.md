# Ordo — AI-ассистент для управления задачами

Telegram бот для голосового и текстового управления задачами. Название: **Ordo** (лат. порядок).

---

## Продуктовое видение

### Проблема которую решает Ordo

У современного человека десятки задач из разных сфер жизни — работа, семья, финансы, здоровье, бытовые дела. Стандартные таск-менеджеры хорошо **хранят** задачи, но не помогают **принимать решения**. Результат — постоянная фоновая тревога, прокрастинация, задачи делаются только когда становятся критически срочными.

**Ordo работает иначе.** Это не инструмент — это советник. Ты выгружаешь всё из головы в доверенную систему, а она сама следит за тем чтобы ничего не провисало и каждый день ты знал что делать.

### Три роли Ordo

**Секретарь** — принимает всё что ты говоришь, записывает, структурирует. Быстро, без трения.

**Планировщик** — понимает контекст задач, расставляет приоритеты, формирует план на день. Без участия пользователя.

**Диспетчер** — каждый день говорит конкретно: вот план на день, делай их, вот почему именно они.

### Ключевой принцип

Пользователь не управляет системой — он живёт, а система работает рядом. Что-то вспомнил — сказал боту. Что-то сделал — сказал боту. Получил план. Вечером очистил голову. Голова свободна.

## Стек

- **Runtime**: Node.js
- **Telegram**: Telegraf
- **База данных**: SQLite (`better-sqlite3`)
- **Speech-to-text**: Groq Whisper API
- **Парсинг задач**: Anthropic Claude API (`claude-sonnet-4-6`)
- **Интеграция**: Notion API (`@notionhq/client` v5, опционально)
- **Тесты**: Jest + in-memory SQLite

## Статусы задач

| Статус | Смысл | Когда |
|--------|-------|-------|
| `todo` | Взял обязательство, надо сделать | По умолчанию при создании |
| `waiting` | Жду чего-то внешнего | Автоопределение Claude или вручную |
| `maybe` | Отложено без конкретной даты | Кнопка [⏭ Отложить] в /review — скрывает на 7 дней |
| `done` | Готово | Вручную или кнопкой в слайдере; очищается еженедельно |
| `deleted` | Удалено (soft delete) | Никогда не удаляется физически |

**Удалены:** `not_started`, `in_progress` — конвертированы в `todo` при миграции.
**Удалён:** `priority` — расставляет AI по контексту, не пользователь вручную.

## Жизненный цикл задач

### Принцип

Создание — без трения. Пользователь не думает "куда это попадёт". Система сама следит и поднимает задачи в нужный момент.

**Единственный момент принятия решения — `/review`.** Там бот спрашивает "что делаем с этим?" — не при создании.

---

### todo — задача без даты (Inbox)

```
Создана → status=todo, planned_for=NULL
    │
    ├─ 0-6 дней → лежит в inbox, не беспокоит
    │
    └─ 7+ дней  → попадает в /review
                  Бот: "Висит N дн. без даты. Когда займёшься?"
                  [Сегодня]  [Завтра]  [На этой неделе]  [📅 Выбрать дату]
                  [⏭ Отложить]  [🗑 Удалить]
```

### todo — задача с датой (В плане)

```
Создана с planned_for → появляется в /plan на эту дату
    │
    └─ Сделана → status=done → еженедельная очистка по воскресеньям
```

### waiting — Тип А (ждём дату)

```
waiting_until задана → бот не трогает задачу пока дата не пришла
    │
    └─ waiting_until < today → попадает в /review
        Бот: "Срок ожидания вышел. Когда займёшься?"
        [Сегодня]  [Завтра]  [На этой неделе]  [📅 Выбрать дату]
        [⏭ Отложить]  [🗑 Удалить]
```

### waiting — Тип Б (ждём событие)

```
waiting_until NULL → ждём неизвестно сколько
    │
    └─ 5+ дней без изменений → попадает в /review
        Бот: "Ждёт уже N дн. Когда займёшься?"
        [Сегодня]  [Завтра]  [На этой неделе]  [📅 Выбрать дату]
        [⏭ Отложить]  [🗑 Удалить]
```

### ⏭ Отложить (maybe)

```
Кнопка [⏭ Отложить] в /review
    │
    └─ status = 'maybe', updated_at = now
       Задача исчезает из /review на 7 дней
       Через 7 дней снова появляется в /review
```

### Повторяющиеся задачи (is_recurring = 1)

```
Создаётся голосом:
"Тренировка каждый вторник и четверг в 20:30"
"Стендап каждый будний в 9:00 и ретро каждую пятницу в 17:00"  ← batch

Хранится в таблице tasks с is_recurring=1.
    │
    ├─ Видна в /tasks с иконкой 🔄 и фильтром "🔄 Повторяющиеся"
    ├─ Детальная карточка показывает расписание и следующую дату
    │
    ├─ Scheduler: в recur_time минус recur_remind_before
    │   → отправляет уведомление с кнопкой [✅ Сделал]
    │   → advanceRecurring() — planned_for = следующая дата по расписанию
    │
    ├─ Пользователь нажимает "Готово" (из плана или карточки)
    │   → advanceRecurring() — переходит на следующий цикл, НЕ ставит done
    │
    ├─ НЕ попадает в /review (ни в "Из плана", ни в "Без даты")
    ├─ НЕ попадает в просроченные (в AI-рекомендациях /plan)
    └─ НЕ удаляется при еженедельной очистке
```

### Еженедельная очистка выполненных задач

```
Каждое воскресенье в 02:00 UTC:
    UPDATE tasks SET status='deleted'
    WHERE status='done' AND is_recurring=0

Повторяющиеся задачи не затрагиваются — они живут в вечном цикле.
```

## Расписание уведомлений

### Автоматические сообщения

| Время | Тип | Условие |
|-------|-----|---------|
| `plan_time` (09:00 по умолчанию) | `/plan` на сегодня | Не отправлялся сегодня + не тихий режим + пользователь активен |
| `review_time` (21:00 по умолчанию) | `/review` | Не отправлялся сегодня + не тихий режим |
| `recur_time - recur_remind_before` | Напоминание о событии | `is_recurring=1`, совпадение времени и дня |
| Воскресенье 02:00 UTC | Очистка выполненных задач | `status='done' AND is_recurring=0` |

### Правила антиспама

- Максимум **2 автосообщения в день** (план + разбор)
- **Тихий режим** (`quiet_until`) — ничего не отправляется
- Пользователь **неактивен 7+ дней** — не отправлять
- Состояние хранится в `notification_log` — работает после перезапуска бота

### Тихий режим

Управляется голосом или через `/settings`:

| Фраза | Действие |
|-------|----------|
| "не беспокой до завтра" | `quiet_until = tomorrow` |
| "не беспокой до воскресенья" | `quiet_until = sunday` |
| "поставь план на 8 утра" | `plan_time = 08:00` |
| "выключи напоминания" | `plan_enabled=0, review_enabled=0` |

## Пользовательский флоу

### /start — Онбординг

```
Новый пользователь (нет задач):
  👋 Привет, [имя]!
  Я — Ordo, твой личный ассистент задач.
  📥 Напиши или скажи задачу — запишу без лишних полей
  📅 Каждое утро пришлю план на день
  🔍 Каждый вечер разберём что зависло
  [⚙️ Настроить время напоминаний]

Возвращающийся пользователь:
  С возвращением, [имя]! + список команд
```

### /plan — План на день

```
1. Автоматически в plan_time (или по команде /plan)
2. Бот сразу показывает план на СЕГОДНЯ:
   ┌────────────────────────────────┐
   │ 📅 План на Пн 14 апр          │
   │                                │
   │ ✅ Запланировано: 2            │
   │ 🤖 Рекомендует AI: 3          │
   │ _Задач нет. Напиши..._        │  ← если пусто
   │                                │
   │ [📋 Запланировано (2)]        │
   │ [🤖 Рекомендации (3)]         │
   │ [📅 Другой день]              │
   └────────────────────────────────┘
3. [📅 Другой день] → календарь для выбора даты
4. В слайдере "Запланировано": [✅ Сделал] [📅 На завтра] [❌ Убрать из плана]
   — для повторяющихся [✅ Сделал] вызывает advanceRecurring, не done
5. В слайдере "Рекомендации":  [➕ Добавить в план]
6. Навигация: ◀️ · 📋 К плану · ▶️   (счётчик "1 из 3" в тексте задачи)
```

### /review — Разбор задач

```
1. Автоматически в review_time (или по команде /review)
   Не отправляется автоматически если нет задач для разбора.
2. Сводный экран:
   ┌────────────────────────────────┐
   │ 🔍 Разбор задач               │
   │                                │
   │ Нужно разобрать: 3            │
   │                                │
   │ [▶️ Начать разбор]            │
   └────────────────────────────────┘
   Или если нечего разбирать:
   "🎉 Всё под контролем!"  [📋 Открыть план на сегодня]

3. Слайдер — каждая задача с причиной попадания:
   🔍 Разбор задач _1 из 3_

   📋 *[название]*
   _Висит 10 дн. без даты_

   Когда займёшься?
   [Сегодня]  [Завтра]  [На этой неделе]
   [📅 Выбрать дату]
   [⏭ Отложить]  [🗑 Удалить]
   [◀️]  [📋 К списку]  [▶️]

4. [На этой неделе] → ближайшая пятница
5. [📅 Выбрать дату] → календарь → ставит planned_for → следующая задача
6. [⏭ Отложить] → status=maybe, задача пропадает из review на 7 дней
7. После последней задачи — итоговый экран:
   ✅ Разбор завершён
   📅 Запланировано: N / ⏭ Отложено: M / 🗑 Удалено: K
   [📋 Открыть план на сегодня]
```

**Пороги попадания в /review** (максимум 5 задач, в порядке приоритета):

| Приоритет | Условие |
|-----------|---------|
| 1 — Waiting (истёк срок) | `status=waiting`, `waiting_until < today` |
| 2 — Waiting (без даты)   | `status=waiting`, `waiting_until IS NULL`, 5+ дней без изменений |
| 3 — Todo без даты        | `status=todo`, `planned_for IS NULL`, 7+ дней без изменений, не повторяющаяся |
| 4 — Maybe                | `status=maybe`, 7+ дней без изменений |

### /tasks — Список задач

```
Фильтры (строка 1): [📁 Категория] [📊 Статус] [📎 Цель]
Фильтры (строка 2): [📅 Сегодня] [📦 Архив] [🔍 Поиск]
Фильтры (строка 3): [🔄 Повторяющиеся]  ← фильтр только recurring задач

Повторяющиеся задачи отображаются с иконкой 🔄 в списке.
Детальная карточка повторяющейся задачи показывает расписание и дату следующего срабатывания.
```

## UX и функционал

### Создание задачи — без трения

1. Пользователь говорит или пишет задачу свободным текстом
2. Бот **немедленно сохраняет** и показывает подтверждение с кнопками `[✏️ Изменить]` `[🗑 Отменить]`
3. Никаких обязательных полей

**Автоопределение waiting:**
Фразы "записался", "заказал", "отправил", "жду", "договорился" → Claude автоматически ставит `status: waiting` с `waiting_reason` и `waiting_until`.

**Инференс категории:**
Claude определяет категорию из контекста задачи. Если подходящая уже есть — использует её. Если нет — создаёт новую ("Здоровье", "Покупки", "Спорт" и т.д.). `null` только если контекст совсем неясен.

## Проактивный ассистент

### Два момента дня

```
📋 plan_time (09:00)   — /plan на сегодня (автоматически)
🔍 review_time (21:00) — /review зависших задач (автоматически)
```

### /plan — AI-приоритизация

Не хранит "список на сегодня" как контейнер — это динамический вид над задачами.

**`getPlanRecommendations`:**
- Задачи с `planned_for` на выбранную дату — первый слайдер
- Из оставшихся: просроченные, waiting с истёкшей датой, давно не трогавшиеся, задачи из активных целей — второй слайдер (AI выбирает до 3)
- Повторяющиеся задачи в просроченные **не попадают**
- `maybe`-задачи **не попадают** в рекомендации (пользователь сознательно отложил)

## Принципы разработки

- **Голос = текст = команда**: любой функционал доступен тремя способами
- **Захват без трения**: никаких обязательных полей при создании задачи
- **AI расставляет приоритеты**: пользователь не должен думать о приоритетах вручную
- **Бот — советник, не диктатор**: предлагает, объясняет, пользователь решает
- **Soft delete**: задачи не удаляются физически
- **Один уровень подзадач**: без рекурсии
- **Интеграции опциональны**: бот работает полностью без Notion
- **Ошибки интеграций не блокируют основной флоу**
- **SQLite → PostgreSQL**: лёгкая миграция при масштабировании
