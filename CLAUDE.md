# Ordo — AI-ассистент для управления задачами

Telegram бот для голосового и текстового управления задачами. Название: **Ordo** (лат. порядок).

---

## Продуктовое видение

### Проблема которую решает Ordo

У современного человека десятки задач из разных сфер жизни — работа, семья, финансы, здоровье, бытовые дела. Стандартные таск-менеджеры хорошо **хранят** задачи, но не помогают **принимать решения**. Результат — постоянная фоновая тревога, прокрастинация, задачи делаются только когда становятся критически срочными.

**Ordo работает иначе.** Это не инструмент — это советник. Ты выгружаешь всё из головы в доверенную систему, а она сама следит за тем чтобы ничего не провисало и каждый день ты знал что делать.

### Три роли Ordo

**Секретарь** — принимает всё что ты говоришь, записывает, структурирует. Быстро, без трения.

**Планировщик** — понимает контекст задач, расставляет приоритеты, формирует план на день. Без участия пользователя.

**Диспетчер** — каждый день говорит конкретно: вот план на день, делай их, вот почему именно они.

### Ключевой принцип

Пользователь не управляет системой — он живёт, а система работает рядом. Что-то вспомнил — сказал боту. Что-то сделал — сказал боту. Утром получил план. Вечером очистил голову. Голова свободна.

## Стек

- **Runtime**: Node.js
- **Telegram**: Telegraf
- **База данных**: SQLite (`better-sqlite3`)
- **Speech-to-text**: Groq Whisper API
- **Парсинг задач**: Anthropic Claude API (`claude-sonnet-4-6`)
- **Интеграция**: Notion API (`@notionhq/client` v5, опционально)
- **Тесты**: Jest + in-memory SQLite

## Статусы задач

### Новая модель (4 статуса)

| Статус | Смысл | Когда |
|--------|-------|-------|
| `todo` | Надо сделать, взял обязательство | По умолчанию при создании |
| `waiting` | Жду чего-то внешнего | Автоопределение или вручную |
| `maybe` | Возможно когда-нибудь | Когда не готов обязаться |
| `done` | Готово | Вручную |
| `deleted` | Удалено | Soft delete |

**Удалены:** `not_started`, `in_progress` — они не отражают реальное поведение пользователя.

**Удалён:** `priority` (high/medium/low) — приоритет расставляет AI на основе контекста, не пользователь вручную.

### Waiting — два типа

**Тип А — ждём дату.** `waiting_until` задана → бот не трогает пока дата не пришла.

**Тип Б — ждём событие.** `waiting_until` не задана → бот поднимает задачу в вечернем разборе через N дней.

### Maybe — отдельная концепция

Задачи которые пользователь не готов удалять но и не обязуется делать. Бот раз в неделю спрашивает: "Ты всё ещё думаешь об этом?" Пользователь переводит в `todo` или удаляет.

## UX и функционал

### Создание задачи — без трения

1. Пользователь говорит или пишет задачу свободным текстом
2. Бот записывает и при необходимости задаёт ОДИН уточняющий вопрос
3. Никаких обязательных полей

**Когда бот спрашивает дедлайн:**
- Задача привязана к событию или человеку → предлагает варианты `[Сегодня]` `[Завтра]` `[На этой неделе]` `[Пропустить]`
- Задача большая без подзадач → предлагает разбить на шаги
- Бытовая мелкая задача → просто создаёт без вопросов

**Автоопределение waiting:**
Фразы "записался", "заказал", "отправил", "жду", "договорился", имя человека + время + контекст встречи → Claude автоматически ставит `status: waiting` с `waiting_reason` и `waiting_until`.

## Фаза 7 — Проактивный ассистент

### Архитектура трёх моментов дня

```
🌅 9:00   /morning  — план на день (проактивно)
🔍 21:00  /review   — разбор накопившегося (проактивно)
📊 пятница /progress — итог недели (по запросу или автоматически)
```

Плюс `/focus` и `/progress` в любой момент по запросу.

---

### /morning — Утренний план

**Как работает:**
Claude анализирует все задачи пользователя и формирует список из 3-5 задач с объяснением почему именно они.

**Алгоритм приоритизации (через Claude API):**
- Дедлайны — самые близкие выше
- Waiting у которых дата истекла
- Задачи которые блокируют другие (зависимости)
- Задачи которые давно не трогались (updated_at старый)
- Контекст планов (свадьба через 2 месяца → задачи из плана Свадьба приоритетнее)

**Пример сообщения:**
```
🌅 Доброе утро!

Вот что предлагаю на сегодня:

1. 💍 Позвонить маме насчёт той
   → свадьба через 2 месяца, это блокирует выбор даты

2. 🏠 Написать подрядчику по ремонту
   → ты ждал его ответа — он ответил вчера

3. ⏸ Проверить доставку с WB
   → срок ожидания вышел

Остальные 22 задачи подождут.

[✅ Берусь]  [✏️ Изменить список]
```

**Поведение кнопок:**
- `[✅ Берусь]` → задачи помечаются как "план на сегодня"
- `[✏️ Изменить список]` → голосом или текстом убрать/добавить задачи

**Правила:**
- Не отправлять если пользователь неактивен 7+ дней
- Не отправлять в тихий режим
- Если добавил задачу вне плана голосом — бот спрашивает "добавить в сегодняшний список?"

---

### /review — Вечерний разбор задач

Отдельная функция от утреннего плана. Другое настроение — рефлексия и очистка, не планирование.

**Что разбирает:**

```
Waiting без даты (Тип Б):
→ висит N дней → спрашивает "готов заняться?"

Waiting с истёкшей датой:
→ выносит на обсуждение → что делаем?

Todo без движения 7+ дней:
→ предлагает: взять в работу / maybe / удалить

Maybe задачи:
→ раз в неделю: всё ещё актуально?
```

**Пример сообщения:**
```
🔍 Вечерний разбор — 3 минуты

⏸ Жду ответа от HR (6 дней, без даты)
   Пора напомнить им?
   [Добавить задачу]  [Ещё жду]  [Закрыть]

⏸ Поговорить с женой об ипотеке (8 дней)
   Готов поговорить?
   [Добавить завтра]  [Ещё не готов]  [В maybe]

📋 Разобраться с релокацией (21 день)
   Давно висит. Что с этим?
   [Разбить на шаги]  [В maybe]  [Удалить]

[Пропустить всё]
```

**Правила:**
- Максимум 5 задач за один разбор
- Каждый вопрос — одно нажатие
- `[Пропустить всё]` — всегда доступен

### Повторяющиеся задачи

Отдельный тип. Не создаётся заново — существует постоянно, напоминает по расписанию.

**Создание голосом:**
```
Ты:  каждый понедельник в 11 вечера у меня
     созвон по работе, напомни за 30 минут

Бот: ✅ Создал повторяющееся напоминание
     🔄 Созвон по работе
     Каждый понедельник в 22:30
```

**Отличие от обычной задачи:**
- После выполнения — сбрасывается на следующий цикл, не удаляется
- Не попадает в основной список задач
- Только напоминание в нужный момент

**Примеры:**
- Созвон каждый понедельник в 23:00 → напомни в 22:30
- Спортзал вторник/четверг → напомни "собери вещи" за час
- Оплата аренды 1-го числа → напомни 30-го

## Принципы разработки

- **Голос = текст = команда**: любой функционал доступен тремя способами
- **Захват без трения**: никаких обязательных полей при создании задачи
- **AI расставляет приоритеты**: пользователь не должен думать о приоритетах вручную
- **Бот — советник, не диктатор**: предлагает, объясняет, пользователь решает
- **Soft delete**: задачи не удаляются физически
- **Один уровень подзадач**: без рекурсии
- **Интеграции опциональны**: бот работает полностью без Notion
- **Ошибки интеграций не блокируют основной флоу**
- **SQLite → PostgreSQL**: лёгкая миграция при масштабировании
