# Ordo — AI-ассистент для управления задачами

Telegram бот для голосового и текстового управления задачами. Название: **Ordo** (лат. порядок).

---

## Продуктовое видение

### Проблема которую решает Ordo

У современного человека десятки задач из разных сфер жизни — работа, семья, финансы, здоровье, бытовые дела. Стандартные таск-менеджеры хорошо **хранят** задачи, но не помогают **принимать решения**. Результат — постоянная фоновая тревога, прокрастинация, задачи делаются только когда становятся критически срочными.

**Ordo работает иначе.** Это не инструмент — это советник. Ты выгружаешь всё из головы в доверенную систему, а она сама следит за тем чтобы ничего не провисало и каждый день ты знал что делать.

### Три роли Ordo

**Секретарь** — принимает всё что ты говоришь, записывает, структурирует. Быстро, без трения.

**Планировщик** — понимает контекст задач, расставляет приоритеты, формирует план на день. Без участия пользователя.

**Диспетчер** — каждый день говорит конкретно: вот план на день, делай их, вот почему именно они.

### Ключевой принцип

Пользователь не управляет системой — он живёт, а система работает рядом. Что-то вспомнил — сказал боту. Что-то сделал — сказал боту. Утром получил план. Вечером очистил голову. Голова свободна.

## Стек

- **Runtime**: Node.js
- **Telegram**: Telegraf
- **База данных**: SQLite (`better-sqlite3`)
- **Speech-to-text**: Groq Whisper API
- **Парсинг задач**: Anthropic Claude API (`claude-sonnet-4-6`)
- **Интеграция**: Notion API (`@notionhq/client` v5, опционально)
- **Тесты**: Jest + in-memory SQLite

## Схема базы данных (SQLite)

### Текущая схема

```sql
users
- id (telegram user id), username, created_at

categories
- id, user_id, name, color

plans
- id, user_id, title, description
- status (active / archived)
- notion_page_id, created_at

tasks
- id, user_id, title, description
- status (todo / waiting / maybe / done / deleted)  ← ОБНОВЛЕНО
- priority (high / medium / low)                    ← планируется убрать
- category_id → categories
- plan_id → plans (nullable)
- due_date (nullable)
- notion_page_id
- waiting_reason TEXT (nullable)
- waiting_until DATE (nullable)
- created_at, updated_at

subtasks
- id, task_id (CASCADE), title, is_done, position, notion_block_id

sync_errors
- id, user_id, message, created_at
```

### Таблицы Фазы 7 (реализованы)

```sql
user_settings
- user_id PRIMARY KEY → users
- morning_time TEXT DEFAULT '09:00'
- evening_time TEXT DEFAULT '21:00'
- timezone TEXT DEFAULT 'Asia/Oral'
- morning_enabled INTEGER DEFAULT 1
- review_enabled INTEGER DEFAULT 1
- quiet_until TEXT NULL
- created_at, updated_at

notification_log
- id, user_id, type, task_id, sent_at, reacted

recurrent_tasks
- id, user_id, title
- event_time TEXT              -- 'HH:MM' время события
- days TEXT                    -- JSON [0-6], null = ежедневно
- day_of_month INTEGER         -- 1-31, null если не ежемесячно
- reminder_before_minutes INTEGER DEFAULT 0
- created_at
```

## Статусы задач

| Статус | Смысл | Когда |
|--------|-------|-------|
| `todo` | Взял обязательство, надо сделать | По умолчанию при создании |
| `waiting` | Жду чего-то внешнего | Автоопределение Claude или вручную |
| `maybe` | Возможно когда-нибудь | Когда не готов обязаться |
| `done` | Готово | Вручную или кнопкой в слайдере |
| `deleted` | Удалено (soft delete) | Никогда не удаляется физически |

**Удалены:** `not_started`, `in_progress` — не отражают реальное поведение.  
**Удалён:** `priority` — расставляет AI по контексту, не пользователь вручную.

## Жизненный цикл задач

### Принцип

Создание — без трения. Пользователь не думает "куда это попадёт". Система сама следит и поднимает задачи в нужный момент.

**Единственный момент принятия решения — `/review`.** Там бот спрашивает "что делаем с этим?" — не при создании.

---

### todo — задача без даты (Inbox)

```
Создана → status=todo, planned_for=NULL
    │
    ├─ 0-2 дня  → лежит в inbox, не беспокоит
    │
    └─ 3+ дня   → попадает в /review (категория "Без даты")
                  Бот: "Лежит без даты. Запланировать или убрать?"
                  [📅 На завтра]  [💭 В maybe]  [🗑 Удалить]
```

### todo — задача с датой (В плане)

```
Создана с planned_for → появляется в /plan на эту дату
    │
    ├─ Дата наступила, не сделана → попадает в /review (категория "Из плана")
    │   Бот: "Было в плане на сегодня."
    │   [✅ Сделал]  [📅 На завтра]  [🗑 Удалить]
    │
    └─ Сделана → status=done, уходит из всех списков
```

### waiting — Тип А (ждём дату)

```
waiting_until задана → бот не трогает задачу пока дата не пришла
    │
    └─ waiting_until < today → попадает в /review (категория "В ожидании")
        Бот: "Срок ожидания вышел."
        [▶️ Взять в работу]  [⏸ Ещё жду]  [❌ Закрыть]
```

### waiting — Тип Б (ждём событие)

```
waiting_until NULL → ждём неизвестно сколько
    │
    └─ 3+ дня без изменений → попадает в /review (категория "В ожидании")
        Бот: "Пора напомнить?"
        [▶️ Взять в работу]  [⏸ Ещё жду]  [❌ Закрыть]
```

### maybe

```
Создана или переведена в maybe
    │
    └─ 7+ дней без изменений → попадает в /review (категория "Может быть")
        Бот: "Всё ещё думаешь об этом?"
        [✅ Да, беру в работу]  [🗑 Удалить]
```

### Повторяющиеся напоминания

```
Отдельная сущность (recurrent_tasks), не задача.
    │
    └─ Scheduler каждую минуту проверяет расписание
        → В нужный момент отправляет напоминание
        → После срабатывания сбрасывается на следующий цикл, не удаляется
```

## Расписание уведомлений

### Автоматические сообщения

| Время | Тип | Условие |
|-------|-----|---------|
| `morning_time` (по умолчанию 09:00) | `/plan` на сегодня | Не отправлялся сегодня + не тихий режим + пользователь активен |
| `evening_time` (по умолчанию 21:00) | `/review` | Не отправлялся сегодня + не тихий режим + есть зависшие задачи |
| По расписанию из `recurrent_tasks` | Напоминание о событии | Совпадение времени и дня недели/месяца |

### Правила антиспама

- Максимум **2 автосообщения в день** (план + разбор)
- **Тихий режим** (`quiet_until`) — ничего не отправляется
- Пользователь **неактивен 7+ дней** — не отправлять
- Состояние хранится в `notification_log` — работает после перезапуска бота

### Тихий режим

Управляется голосом или через `/settings`:

| Фраза | Действие |
|-------|----------|
| "не беспокой до завтра" | `quiet_until = tomorrow` |
| "не беспокой до воскресенья" | `quiet_until = sunday` |
| "поставь план на 8 утра" | `morning_time = 08:00` |
| "выключи напоминания" | `morning_enabled=0, review_enabled=0` |

## Пользовательский флоу

### Утро — /plan

```
1. Автоматически в morning_time (или по команде /plan)
2. Бот показывает выбор даты: Сегодня / Завтра / +2 / +3
3. После выбора: "⏳ Анализирую задачи на Пн 14 апр..."
4. Появляется сводная карточка:
   ┌────────────────────────────────┐
   │ 📅 План на Пн 14 апр          │
   │                                │
   │ ✅ Запланировано: 2            │
   │ 🤖 Рекомендует AI: 3          │
   │                                │
   │ [📋 Запланировано (2)]        │
   │ [🤖 Рекомендации (3)]         │
   └────────────────────────────────┘
5. Пользователь открывает нужную категорию → слайдер по задачам
6. В слайдере "Запланировано": [✅ Сделал] [📅 На завтра] [❌ Убрать из плана]
7. В слайдере "Рекомендации":  [➕ Добавить в план]
8. Навигация: ◀️ · 📋 К плану · ▶️   (счётчик в тексте задачи: "1 из 3")
```

### Вечер — /review

```
1. Автоматически в evening_time (или по команде /review)
2. Бот считает зависшие задачи по 4 категориям и показывает сводку:
   ┌────────────────────────────────┐
   │ 🔍 Разбор задач               │
   │                                │
   │ 📅 Из плана на сегодня: 1     │
   │ ⏸ В ожидании: 2              │
   │ 📋 Без даты: 3                │
   │ 💭 Может быть: 1              │
   │                                │
   │ [📅 Из плана (1)]             │
   │ [⏸ В ожидании (2)]           │
   │ [📋 Без даты (3)]             │
   │ [💭 Может быть (1)]           │
   └────────────────────────────────┘
3. Пользователь открывает нужную категорию → слайдер по задачам
4. Для каждой задачи — кнопки действий зависят от категории:
   - "Из плана":   [✅ Сделал] [📅 На завтра] [🗑 Удалить]
   - "В ожидании": [▶️ Взять в работу] [⏸ Ещё жду] [❌ Закрыть]
   - "Без даты":   [📅 На завтра] [💭 В maybe] [🗑 Удалить]
   - "Может быть": [✅ Да, беру в работу] [🗑 Удалить]
5. Навигация: ◀️ · 📋 К списку · ▶️   (счётчик в тексте задачи)
6. После разбора всех задач категории — "✅ Разобрано!", кнопка возврата к сводке
```

## UX и функционал

### Создание задачи — без трения

1. Пользователь говорит или пишет задачу свободным текстом
2. Бот записывает и при необходимости задаёт ОДИН уточняющий вопрос
3. Никаких обязательных полей

**Когда бот спрашивает дедлайн:**
- Задача привязана к событию или человеку → предлагает варианты `[Сегодня]` `[Завтра]` `[На этой неделе]` `[Пропустить]`
- Задача большая без подзадач → предлагает разбить на шаги
- Бытовая мелкая задача → просто создаёт без вопросов

**Автоопределение waiting:**
Фразы "записался", "заказал", "отправил", "жду", "договорился", имя человека + время + контекст встречи → Claude автоматически ставит `status: waiting` с `waiting_reason` и `waiting_until`.

## Проактивный ассистент

### Три момента дня

```
🌅 morning_time (09:00)  — /plan на сегодня (автоматически)
🔍 evening_time (21:00)  — /review зависших задач (автоматически)
📊 по запросу            — /progress прогресс за день/неделю
```

### /plan — План на день

Работает как выбор даты + AI-рекомендации. Не хранит "список на сегодня" как контейнер — это динамический вид над задачами.

**AI-приоритизация (через Claude API):**
- Задачи с `planned_for` на выбранную дату — первый слайдер
- Из оставшихся: waiting с истёкшей датой, давно не трогавшиеся, задачи из активных целей — второй слайдер (рекомендации)
- Лимит рекомендаций: 3-5 задач

**Слайдер "Запланировано":** `[✅ Сделал]` `[📅 На завтра]` `[❌ Убрать из плана]`  
**Слайдер "Рекомендации":** `[➕ Добавить в план]`

### /review — Вечерний разбор

Настроение рефлексии и очистки, не планирования. Показывает только зависшее.

**Пороги попадания в /review:**

| Категория | Условие |
|-----------|---------|
| 📅 Из плана | `planned_for = today` и статус не `done`/`deleted` |
| ⏸ В ожидании | `waiting_until < today` ИЛИ `waiting_until IS NULL` и 3+ дней без изменений |
| 📋 Без даты | `todo`, нет `planned_for`, 3+ дней без изменений |
| 💭 Может быть | `maybe`, 7+ дней без изменений |

### /progress — Прогресс

```
📊 Прогресс

✅ Сегодня: 2 задачи
✅ За неделю: 11 задач

💍 Свадьба    ████░░░░  5/14
🏠 Ремонт     ██░░░░░░  3/10

⚠️ Давно без движения:
• Разобраться с налогами (12 дн.)
```

### Повторяющиеся напоминания

Отдельная сущность (`recurrent_tasks`). Не задача — существует постоянно, после срабатывания сбрасывается на следующий цикл. Не попадает в `/tasks` и `/review`.

**Создание голосом:**
```
"Каждый понедельник в 23:00 созвон, напомни за 30 минут"
→ event_time=23:00, days=[1], reminder_before_minutes=30
```

**Управление:** `/reminders` — список с кнопками удаления.

## Принципы разработки

- **Голос = текст = команда**: любой функционал доступен тремя способами
- **Захват без трения**: никаких обязательных полей при создании задачи
- **AI расставляет приоритеты**: пользователь не должен думать о приоритетах вручную
- **Бот — советник, не диктатор**: предлагает, объясняет, пользователь решает
- **Soft delete**: задачи не удаляются физически
- **Один уровень подзадач**: без рекурсии
- **Интеграции опциональны**: бот работает полностью без Notion
- **Ошибки интеграций не блокируют основной флоу**
- **SQLite → PostgreSQL**: лёгкая миграция при масштабировании
