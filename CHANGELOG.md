# Changelog — TaskBot

Все значимые изменения фиксируются здесь.
Формат: [версия] — дата — что изменилось.

---

## [0.3.0] — 2026-03-17

### Добавлено
- Notion интеграция восстановлена и рефакторена в полноценный адаптер
- Полная синхронизация задач: создание, редактирование, удаление, смена статуса
- Синхронизация планов: создание, редактирование, архив, восстановление, удаление
- Синхронизация подзадач через `notion_block_id` (точечное обновление)
- `/settings` — статус подключения Notion + кнопка массовой синхронизации
- Предупреждение пользователю при ошибке синхронизации
- Тесты: Jest + in-memory SQLite, 81 тест

### Изменено
- `isConfigured()` проверяет наличие `NOTION_TOKEN` + `NOTION_DATABASE_ID`
- Notion API версия: `2022-06-28` (стабильная, без multiple data sources)
- `notion_page_id` сохраняется в таблице `tasks`
- `plan_notion_page_id` сохраняется в таблице `plans`
- `notion_block_id` сохраняется в таблице `subtasks`

### Исправлено
- Notion API ошибка "multiple data sources" при создании задачи

---

## [0.2.0] — 2026-03-15

### Добавлено
- Подзадачи: создание из голоса/текста, тогл, добавление, редактирование, удаление
- AI-предложение подзадач
- Голосовое/текстовое управление всем функционалом (manage_task, query_tasks, manage_plan, manage_tasks_bulk)
- Fuzzy-поиск задач и планов (нормализация, word-by-word matching)
- Групповые операции: "все задачи", "первые N", "последние N", "половина" + фильтры
- AI-разбивка цели на план + задачи с подзадачами
- Архив планов: архивировать, восстановить, удалить
- Задачи архивных планов скрыты в `/tasks` по умолчанию, открываются через 📦 Архив
- Привязка задачи к плану из детальной карточки
- `/settings` команда (базовая)

### Изменено
- Структура проекта: выделены `handlers/`, `formatters.js`, `keyboards.js`, `renderers.js`, `fuzzy.js`, `state.js`, `helpers.js`
- `pendingTasks` хранит `{ task, editingField }` вместо просто `task`
- Категории: если не определена — дефолт "Общее" без лишнего вопроса

## [0.1.0] — 2026-03-10

### Добавлено
- Базовый Telegram бот на Telegraf
- Голосовой ввод через Groq Whisper API
- Парсинг задачи через Anthropic Claude API (название, описание, дата, категория, приоритет, план)
- SQLite как основное хранилище (`better-sqlite3`)
- Таблицы: `users`, `categories`, `plans`, `tasks`, `subtasks`
- Команды: `/start`, `/help`, `/add`, `/tasks`, `/today`, `/plans`
- `/tasks` — список с фильтрами (категория, план, поиск, архив), тап → карточка
- `/plans` — список с прогрессом, создание, редактирование
- Редактирование любого поля задачи через ✏️ Изменить
- Смена статуса и удаление задачи из списка
- Детальная карточка задачи
- Привязка задачи к плану при создании
- Динамические категории из SQLite
- Опциональная синхронизация в Notion (первая версия)
