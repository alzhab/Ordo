## Какую проблему решает Ordo

Стандартные таск-менеджеры требуют открыть приложение, нажать кнопку "создать", заполнить поля. Это прерывает поток — особенно когда задача возникла в голове на ходу.

**Ordo работает иначе:**

- Думаешь задачу → говоришь или пишешь в Telegram → она уже в списке
- Бот сам извлекает дату, приоритет, категорию, подзадачи
- Если задача подразумевает ожидание ("отправил", "записался") — статус ставится автоматически
- Управление через голос: "удали задачу молоко", "переведи отчёт в работу"

## Как это выглядит

**Создание задачи:**
```
Ты:   Записаться к стоматологу на следующей неделе, приоритет высокий

Бот:  📋 Записаться к стоматологу
      📅 Дата: 31 марта
      🔴 Приоритет: высокий

      [✅ Создать]  [✏️ Изменить]  [❌ Отмена]
```

**Автоопределение ожидания:**
```
Ты:   Отправил резюме в компанию X, жду ответа до пятницы

Бот:  📋 Отправил резюме в компанию X
      ⏸ Статус: В ожидании
      💬 Причина: жду ответа
      📅 До: 28 марта
```

**Групповые операции:**
```
Ты:   Первые 3 задачи — в работу

Бот:  ✅ 3 задачи переведены в работу
```

**AI-план:**
```
Ты:   Хочу запустить свой интернет-магазин

Бот:  📁 План: Запуск интернет-магазина
      ├─ Выбрать платформу (шаги: сравнить Shopify/WooCommerce, ...)
      ├─ Настроить домен и хостинг
      ├─ Добавить товары
      ├─ Настроить оплату
      └─ Запустить рекламу
```

| Компонент | Технология |

| База данных | SQLite (`better-sqlite3`) |
| Notion sync | Notion API (опционально) |
| Хостинг | Railway / VPS |
