# Интеграция с Notion

Ordo полностью работает без Notion. Если ты уже используешь Notion — подключи, и задачи будут автоматически появляться там при создании и обновлении.

**Что даёт интеграция:**
- Задачи и планы в удобной таблице Notion
- Подзадачи синхронизируются как чекбоксы
- Статус `waiting` с причиной и датой виден в Notion
- Ошибки синхронизации не блокируют бота — видны в `/settings`

---

## Что синхронизируется

| Поле | Задачи | Планы |
|------|--------|-------|
| Название | ✅ | ✅ |
| Статус | ✅ | ✅ |
| Дата | ✅ | — |
| Описание | ✅ | ✅ |
| Категория | ✅ | — |
| Причина ожидания | ✅ | — |
| Дата ожидания | ✅ | — |
| Подзадачи | ✅ (чекбоксы) | — |

---

## Шаг 1 — Создай интеграцию Notion

1. Зайди на [notion.so/my-integrations](https://www.notion.so/my-integrations)
2. **+ New integration** → укажи название (например `Ordo`) и выбери свой workspace
3. Скопируй **Internal Integration Token** — это твой `NOTION_TOKEN`

---

## Шаг 2 — Подготовь базы данных

Нужны две базы: для задач и для планов.

### База задач

Создай новую базу в Notion со свойствами:

| Свойство | Тип |
|----------|-----|
| Name | Title |
| Status | Status |
| Category | Select |
| Due Date | Date |
| Description | Text |
| Waiting Reason | Text |
| Waiting Until | Date |

Для поля **Status** добавь варианты:
- `Todo` → группа "Not started"
- `Waiting` → группа "In progress"
- `Maybe` → группа "Not started"
- `Done` → группа "Done"

### База планов

| Свойство | Тип |
|----------|-----|
| Name | Title |
| Status | Select (`active`, `archived`) |
| Description | Text |

---

## Шаг 3 — Подключи интеграцию к базам

В каждой базе: нажми `...` → **Connections** → найди свою интеграцию и добавь.

Сделай это для обеих баз — задач и планов.

---

## Шаг 4 — Скопируй ID баз

Открой базу в браузере. ID — часть URL между последним `/` и `?`:

```
https://notion.so/workspace/7d4939ac3f4c836ca99201ae06bafcae?v=...
                             ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                             это и есть ID базы
```

---

## Шаг 5 — Добавь переменные окружения

```env
NOTION_TOKEN=ntn_xxxxxxxxxxxxxx
NOTION_DATABASE_ID=7d4939ac...       # ID базы задач
NOTION_PLANS_DATABASE_ID=324939ac... # ID базы планов
```

После перезапуска бота статус интеграции появится в `/settings`.

---

## Проверка

Напиши в бот любую задачу. Открой Notion — задача должна появиться в базе.

Если что-то не синхронизировалось — `/settings` → раздел "Ошибки синхронизации" покажет детали.
