# Алиса + Ordo: сценарии использования

## Что уже работает

- Привязка аккаунта (6-значный код)
- Захват задачи (текст → AI parser)
- "Мои задачи" → список (до 5)
- "План на сегодня" → задачи с `planned_for = today`
- "Задача 3 готова" → mark done по номеру

---

## Ключевой инсайт про колонку

**Колонка = нет экрана, нет рук, нет кнопок.** Это радикально другой интерфейс. Всё что работает в Telegram через inline-клавиатуры — здесь не работает. Зато колонка делает то что телефон не делает: **ты говоришь не поднимая руки**, в кухне, спальне, ванной.

---

## Идеальные сценарии

### 1. Захват без рук (уже работает — это core value)
Кухня, готовишь: *"Алиса, запиши в Орdo: позвонить врачу завтра"*
Машина (если есть колонка): *"купить подарок для мамы, срочно"*

Это главный сценарий. Всё остальное — дополнения.

### 2. Утренний брифинг (не реализован, высокий приоритет)
*"Алиса, что у меня сегодня?"*
→ Алиса называет **топ-3 задачи** с коротким объяснением почему именно они (как `/plan` + AI-рекомендации)

Сейчас `open_plan` просто берёт `plannedToday` — без AI-приоритизации. Нужно подключить `getPlanRecommendations`.

### 3. Отметка выполненных по названию (не реализован)
Сейчас: *"задача 3 готова"* — нужно помнить номер, что невозможно.
Нужно: *"Алиса, отметь купить молоко"* → fuzzy match по title.

Это критичный gap — колонка без экрана, номер задачи никто не помнит.

### 4. Быстрый вопрос "что самое важное" (не реализован)
*"Алиса, что мне сейчас делать?"*
→ Одна задача. Не список — одна. AI выбирает по контексту.

Для колонки список из 5 — много. Ухо удерживает 2-3 максимум.

### 5. Категорийный захват (не реализован)
*"Алиса, добавь в покупки: молоко, хлеб, яйца"*
→ `create_tasks_batch` + инференс категории "покупки"

Это уже частично есть через batch, но без явного указания категории голосом.

### 6. Вечерний итог (не реализован)
*"Алиса, что я не сделал сегодня?"*
→ Задачи с `planned_for = today` и `status != done`

---

## Что через колонку делать НЕ нужно

- Управление целями — слишком сложно без экрана
- Настройки, /review — требуют визуального выбора
- Редактирование задач — без подтверждения на экране опасно
- Длинные списки — ухо не удержит больше 3-4 позиций

---

## Что стоит доработать прямо сейчас

Три вещи дадут 80% ценности:

1. **Fuzzy match по названию** вместо/вместе с номером — иначе "отметить выполненной" бесполезно без экрана
2. **Ответы покороче** — сейчас до 5 задач, лучше топ-3 + "и ещё N"
3. **AI-приоритизация в утреннем брифинге** — подключить `getPlanRecommendations` к Алисе
